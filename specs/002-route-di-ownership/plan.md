# Implementation Plan: Route DI Ownership

**Branch**: `002-route-di-ownership` | **Date**: 2026-06-28 | **Spec**: [spec.md](./spec.md)

**Input**: Feature specification from `/specs/002-route-di-ownership/spec.md`

## Summary

Convert `auth_routes.ts` from a factory function that receives a `LoginUseCase`
parameter to a standard Fastify plugin that composes its own dependencies internally.
Strip `fastify-routes.ts` of all domain imports — it becomes a thin router that only
calls `app.register`. Update `auth_routes.spec.ts` to use Vitest module mocking
(`vi.mock`) instead of constructor injection. Zero behavioural change to endpoints.

## Phase 0: Research

### Decision: `vi.mock` for route test isolation

**Problem**: After the refactor, `auth_routes.ts` instantiates `UserRepository` and
`HotelRepository` directly at the top of the plugin body. Test files can no longer
pass in-memory fakes via a constructor parameter.

**Options evaluated**:

| Option | Verdict |
|---|---|
| Keep factory pattern, inject fakes via param | Rejected — violates the architectural rule being corrected |
| Real database in route tests | Rejected — violates Principle VII (Testability by Design) |
| `vi.mock` to replace repository modules | **Selected** — standard Vitest pattern; no new dependencies |
| Fastify `decorate` to inject a test repo | Rejected — requires route to read from `app`, not from its own scope |

**`vi.mock` approach**:

```ts
// auth_routes.spec.ts
import { vi, beforeEach } from 'vitest'

// Executed before module graph is loaded — replaces the real implementations
vi.mock('@/db/repositories', () => ({
  UserRepository: vi.fn().mockImplementation(() => userRepo),
  HotelRepository: vi.fn().mockImplementation(() => hotelRepo),
}))
vi.mock('@/db/client', () => ({ db: {} }))
```

`vi.mock` is hoisted by Vitest to run before any imports, so the route plugin will
receive the in-memory instances when it calls `new UserRepository(db)`.

**Rationale**: No new packages. No architectural compromise. The route still reads
`new UserRepository(db)` — the test replaces what that constructor produces.

## Phase 1: Design

### `auth_routes.ts` — target shape

```ts
export async function auth_routes(app: FastifyTypedInstance) {
  const login_use_case = new LoginUseCase(
    new UserRepository(db),
    new HotelRepository(db),
    new BcryptPasswordHasher(),
  )

  app.get('/auth/me', { onRequest: [app.authenticate] }, ...)
  app.post('/auth/login', ..., async (request, reply) => {
    await login_use_case.execute(request.body)
    ...
  })
}
```

- Named `async function` export — no currying, no factory wrapper
- Imports: `db` from `@/db/client`; `UserRepository`, `HotelRepository` from
  `@/db/repositories`; `BcryptPasswordHasher` from `@/lib/bcrypt_password_hasher`;
  `LoginUseCase` from `@/core/usecases/login_use_case`
- All handler logic remains identical

### `fastify-routes.ts` — target shape

```ts
import type { FastifyTypedInstance } from '@/types/fastify'
import { auth_routes } from '@/routes/auth/auth_routes'

export function setRoutes(app: FastifyTypedInstance) {
  app.register(auth_routes)
}
```

- Two imports only: `FastifyTypedInstance` (type) and `auth_routes`
- Zero domain objects

### `auth_routes.spec.ts` — test isolation strategy

```ts
import { vi } from 'vitest'
import { InMemoryUserRepository } from '@/db/repositories/users/in-memory/...'
import { InMemoryHotelRepository } from '@/db/repositories/hotels/in-memory/...'

let userRepo: InMemoryUserRepository
let hotelRepo: InMemoryHotelRepository

vi.mock('@/db/client', () => ({ db: {} }))
vi.mock('@/db/repositories', () => ({
  UserRepository: vi.fn().mockImplementation(() => userRepo),
  HotelRepository: vi.fn().mockImplementation(() => hotelRepo),
}))

// beforeEach: create new repo instances and seed them
// Tests: same four scenarios as current spec (login OK/fail, /me auth/unauth)
```

**Note**: `vi.mock` closures capture the outer `userRepo`/`hotelRepo` variables by
reference. Assigning new instances in `beforeEach` updates what the mocked constructor
returns on next call.
